# 未实现 RPC 字段分析与实现建议

**项目**: Transmission Web Manager\
**分析日期**: 2026-05-03\
**验证实例**: Transmission 4.1.1 @ 127.0.0.1:9091（历史验证）\
**参考文档**: Transmission 4.1.x 官方 RPC 规范

***

## 一、字段分类总览

| 分类 | 字段数 | 建议 |
|------|--------|------|
| 建议实现 | 12 | 有明确用户价值，前端可展示或操作 |
| 可选实现 | 8 | 有一定价值但优先级低 |
| 不建议实现 | 33 | 价值极低、仅调试用、或已有替代方案 |

***

## 二、建议实现的字段（12 个）

### 2.1 `queue_position` — 队列位置

| 项目 | 说明 |
|------|------|
| 4.1.x 字段 | `queue_position` |
| 老版本字段 | `queuePosition` |
| 类型 | `number` |
| 所在方法 | `torrent_get`（响应字段）、`torrent_set`（设置参数） |
| 已验证 | ✅ 本地实例确认返回正确数值 |
| 作用 | 表示种子在下载队列中的排序位置（0 = 最高优先级） |
| 前端价值 | **高**。用户可直观看到种子在队列中的位置，配合已有的队列移动操作（上移/下移/置顶/置底）形成完整功能闭环 |
| 实现位置 | 列表页增加一列显示队列序号；详情页基本信息区展示 |
| 操作方式 | 已有 `queueMoveTop/Up/Down/Bottom()` 方法，只需在 UI 上展示当前值即可 |
| 实现难度 | ⭐ 极低，仅展示 |

### 2.2 `availability` — 分片可用性

| 项目 | 说明 |
|------|------|
| 4.1.x 字段 | `availability` |
| 老版本字段 | `availability` |
| 类型 | `array<number>`（每个元素表示该分片在 swarm 中可获取的副本数） |
| 所在方法 | `torrent_get` |
| 已验证 | ✅ 官方文档确认，RPC 17+ 引入 |
| 作用 | 数组长度 = 分片数，每个值表示该分片在连接到的 peers 中有多少份。值为 0 表示该分片当前无法获取 |
| 前端价值 | **高**。类似 qBittorrent 的"分片可用性"热力图，可直观展示哪些分片稀缺、哪些充足，对做种和下载策略有指导意义 |
| 实现位置 | 分片 tab 中，与现有的点状分布图结合，用颜色深浅表示可用性高低 |
| 操作方式 | 纯展示，无需操作 |
| 实现难度 | ⭐⭐ 中等，需要复用 canvas 绘制逻辑，用颜色映射可用性值 |

## 四、不建议实现的字段（33 个）

### 4.1 Tracker 调试字段（17 个）

以下字段属于 tracker 连接的底层调试信息，仅在排查 tracker 连接问题时才有用：

| 字段 | 所在 | 说明 | 不实现理由 |
|------|------|------|-----------|
| `announce_state` | `trackers[]` | 宣告状态（0=未宣告, 1=正在宣告, 2=已宣告, 3=宣告失败） | 已有 `lastAnnounceResult` 和 `nextAnnounceTime` 足够 |
| `has_announced` | `trackers[]` | 是否已进行过宣告 | 通过 `lastAnnounceTime` 可推断 |
| `last_announce_start_time` | `trackers[]` | 上次宣告开始时间 | 时间戳细节，用户不需要 |
| `has_scraped` | `trackers[]` | 是否进行过刮取 | 通过 `lastScrapeTime` 可推断 |
| `last_scrape_start_time` | `trackers[]` | 上次刮取开始时间 | 时间戳细节 |
| `last_scrape_result` | `trackers[]` | 上次刮取结果字符串 | 错误信息已够用 |
| `last_scrape_succeeded` | `trackers[]` | 上次刮取是否成功 | 通过 `lastScrapeTime` 和结果可推断 |
| `last_scrape_time` | `trackers[]` | 上次刮取时间 | 已有 `lastScrapeTime` |
| `last_scrape_timed_out` | `trackers[]` | 上次刮取是否超时 | 调试细节 |
| `next_scrape_time` | `trackers[]` | 下次刮取时间 | 用户不关心刮取调度 |
| `scrape_state` | `trackers[]` | 刮取状态码 | 调试用 |
| `scrape` | `trackers[]` | 刮取 URL | 通常与 announce URL 同域，单独展示意义不大 |
| `is_backup` | `trackers[]` | 是否为备用 tracker | 4.1.x 的 tier 机制已替代 backup 概念 |
| `from_cache` | `tracker_stats[]` | 来自缓存的 peer 数 | 已归入"建议实现"类别 |
| `download_count` | `tracker_stats[]` | 下载完成次数 | 已归入"建议实现"类别 |
| `downloader_count` | `tracker_stats[]` | 当前下载者数 | 已归入"建议实现"类别 |
| `seeder_count` / `leecher_count` | `tracker_stats[]` | 做种者/下载者数 | **已实现** |

> **总结**: 这些字段适合在 Transmission 桌面客户端的"调试窗口"中查看，但 Web 管理界面面向的是日常管理操作，不需要暴露这么多底层细节。
