## 二、未实现字段详细分析

### 2.1 torrent_get: `bytes_completed`

| 属性 | 值 |
|---|---|
| 旧协议字段名 | `bytesCompleted` |
| 4.1.x 字段名 | `bytes_completed` |
| 类型 | array (number[]) |
| 官方定义 | 各文件已完成字节数组，长度等于文件数量 |
| **本地实测** | ✅ 真实存在，返回 `[2992299819]` 等数值 |
| 与已有功能的关系 | `files[].bytesCompleted` 已在文件列表中单独返回每个文件的完成字节数；`fileStats[].bytesCompleted` 同样返回此值 |

**分析**: 此字段是 `files` 和 `fileStats` 子对象中 `bytesCompleted` 的顶层汇总版本，作为独立数组返回。项目已通过 `t.files[i].bytes_completed` 在文件列表中逐个显示每个文件的完成字节数。

**UI 实现建议**: ❌ **无需实现**。功能已通过 `files[].bytes_completed` 完全覆盖，顶层汇总数组仅是冗余的另一种数据组织形式，无额外信息价值。

---

### 2.2 torrent_get: `webseeds_ex`

| 属性 | 值 |
|---|---|
| 旧协议字段名 | 无（JSON-RPC 2.0 新增） |
| 4.1.x 字段名 | `webseeds_ex` |
| 类型 | array of objects |
| 官方定义 | Web 种子扩展信息，包含 `url`、`is_downloading`、`download_bytes_per_second` |
| **本地实测** | ⚠️ 当前实例无 Web 种子种子，无法实测返回值。但官方文档明确此字段为 4.2.0+ 新增 |
| 与已有功能的关系 | `webseeds` (旧字段，返回 string[]) 已在 DETAIL_FIELDS 中请求并在详情页显示 |

**子字段详情**:

| 子字段 | 类型 | 说明 | 本地实测 |
|---|---|---|---|
| `url` | string | Web 种子 URL | 与 `webseeds[]` 中的 URL 相同 |
| `is_downloading` | boolean | 是否正在从此 Web 种子下载 | 未实测 |
| `download_bytes_per_second` | number | 从此 Web 种子的下载速度 (B/s) | 未实测 |

**分析**: `webseeds_ex` 是 `webseeds` 的扩展版本，额外提供了每个 Web 种子的实时下载状态和速度。`webseeds` 已被官方标记为 DEPRECATED（4.2.0），未来将被 `webseeds_ex` 替代。但当前 4.1.x 版本中 `webseeds_ex` 尚未正式可用。

**UI 实现建议**: ⏳ **暂缓实现**。当前 4.1.x 版本此字段尚未稳定，建议在 4.2.0 正式发布后实现。实现时可替换现有 `webseeds` 显示，在 Web 种子列表中增加下载状态和速度列。

---

### 2.3 torrent_get: `sequential_download_from_piece`

| 属性 | 值 |
|---|---|
| 旧协议字段名 | `sequential_download_from_piece` |
| 4.1.x 字段名 | `sequential_download_from_piece` |
| 类型 | number |
| 官方定义 | 顺序下载起始分片索引，配合 `sequential_download` 使用 |
| **本地实测** | ✅ 真实存在，默认返回 `0`。设置后可正确读取（实测设置 `5` 后读取返回 `5`） |

**分析**: 此字段允许用户指定从哪个分片开始顺序下载，适用于需要优先下载文件中间部分的场景（如视频文件从特定位置开始播放）。项目已实现 `sequential_download` 开关，但未实现起始分片设置。

**UI 实现建议**: 🟡 **低优先级**。可在设置 Tab 的顺序下载开关旁增加一个"起始分片"输入框，仅在 `sequential_download` 为 `true` 时显示。实现方案：
1. 在 [ui-detail-extras.js](file:///home/qq/transmission_web_manager/js/ui-detail-extras.js) 的 `renderSettings` 中增加输入框
2. 在 [rpc.js](file:///home/qq/transmission_web_manager/js/rpc.js) 的 `_torrentSetKeyMap` 中添加 `sequential_download_from_piece` 映射
3. 需在 DETAIL_FIELDS 中添加 `sequential_download_from_piece` 字段请求

### 2.4 torrent_get files 子对象: `begin_piece` / `end_piece`

| 属性 | 值 |
|---|---|
| 旧协议字段名 | `begin_piece` / `end_piece` |
| 4.1.x 字段名 | `begin_piece` / `end_piece` |
| 类型 | number |
| 官方定义 | 文件对应的起始/结束分片索引 |
| **本地实测** | ✅ 真实存在。实测文件0返回 `begin_piece: 0, end_piece: 742` |

**分析**: 这两个字段标识每个文件在分片位图中的范围，可以用于在分片可视化中高亮显示某个文件对应的分片区域。项目已有分片 Tab（pieces）显示分片完成度和可用性。

**UI 实现建议**: 🟡 **低优先级**。可用于增强分片 Tab 的交互体验：点击文件列表中的某个文件时，在分片可视化中高亮该文件对应的分片范围。实现方案：
1. 在 [rpc.js](file:///home/qq/transmission_web_manager/js/rpc.js) 的 `_camelToSnakeMap` 中添加 `begin_piece` 和 `end_piece` 映射
2. 在 [ui-detail.js](file:///home/qq/transmission_web_manager/js/ui-detail.js) 的 `_renderFiles` 中存储每个文件的分片范围
3. 在 [ui-detail-extras.js](file:///home/qq/transmission_web_manager/js/ui-detail-extras.js) 的分片渲染中支持高亮指定范围

### 2.5 session_get: `units`

| 属性 | 值 |
|---|---|
| 旧协议字段名 | `units` |
| 4.1.x 字段名 | `units` |
| 类型 | object |
| 官方定义 | 速度/大小/内存的单位配置 |
| **本地实测** | ✅ 真实存在 |

**实测返回值**:
```json
{
  "units": {
    "memory-bytes": 1024,
    "memory-units": ["B", "KiB", "MiB", "GiB", "TiB"],
    "size-bytes": 1000,
    "size-units": ["B", "kB", "MB", "GB", "TB"],
    "speed-bytes": 1000,
    "speed-units": ["B/s", "kB/s", "MB/s", "GB/s", "TB/s"]
  }
}
```

**分析**: `units` 对象提供了 Transmission 使用的单位换算规则。当前项目在 [utils.js](file:///home/qq/transmission_web_manager/js/utils.js) 中硬编码了单位换算逻辑（`formatBytes`、`formatSpeed` 等函数使用 1000/1024 进制和固定单位标签）。使用 `units` 字段可以确保前端显示与 Transmission 后端/CLI 的单位完全一致。

**UI 实现建议**: 🟡 **低优先级**。当前硬编码的单位换算已能正常工作，且 Transmission 的单位体系不太可能频繁变更。如需实现，方案为：
1. 在 [utils.js](file:///home/qq/transmission_web_manager/js/utils.js) 中将 `formatBytes`/`formatSpeed` 改为从 session 数据读取单位配置
2. 缓存 session 中的 units 配置，避免每次格式化都查找

## 六、附录：本地实测命令与结果

### A. bytes_completed 测试
```bash
curl -s -d '{"method":"torrent-get","arguments":{"fields":["id","name","bytesCompleted"]}}'
# 返回: "bytesCompleted": [2992299819]  (单文件种子)
# 返回: "bytesCompleted": [1088972225, 38690, 840]  (多文件种子)
```

### B. sequential_download_from_piece 测试
```bash
# 设置
curl -s -d '{"method":"torrent-set","arguments":{"ids":[3],"sequential_download":true,"sequential_download_from_piece":5}}'
# 读取
curl -s -d '{"method":"torrent-get","arguments":{"ids":[3],"fields":["sequential_download","sequential_download_from_piece"]}}'
# 返回: {"sequential_download":true,"sequential_download_from_piece":5}
```

### C. begin_piece / end_piece 测试
```bash
curl -s -d '{"method":"torrent-get","arguments":{"fields":["id","name","files"]}}'
# files 子对象中包含: "begin_piece": 0, "end_piece": 742
```

### D. units 测试
```bash
curl -s -d '{"method":"session-get","arguments":{"fields":["units"]}}'
# 返回: units 对象包含 speed/size/memory 的单位标签和进制
```

### E. tracker_list 增删改测试
```bash
# 添加 Tracker (新 tier)

# 结果: 成功，open.stealth.si 被替换为 explodie.org

# 移除 Tracker

```
